# 🔐 Настройка пароля для админ-панели

Инструкция по установке и изменению пароля для доступа к админ-панели.

---

## 🚨 ВАЖНО: Измени пароль по умолчанию!

**Текущий пароль:** `password`

Это **небезопасно**! Обязательно измени пароль перед использованием в production.

---

## 🔧 Как изменить пароль

### Способ 1: Автоматический (рекомендуется)

```bash
# Сгенерируй хеш для нового пароля
npx tsx generate-password-hash.ts ВАШ_НОВЫЙ_ПАРОЛЬ

# Скопируй сгенерированный хеш
# Открой src/pages/Admin.tsx
# Найди строку: const ADMIN_PASSWORD_HASH = '...'
# Замени хеш на свой
```

### Способ 2: Вручную

1. **Открой онлайн SHA-256 генератор:**
   - https://emn178.github.io/online-tools/sha256.html
   - Или любой другой SHA-256 генератор

2. **Введи свой пароль** и получи хеш

3. **Открой файл:** `src/pages/Admin.tsx`

4. **Найди строку:**
   ```typescript
   const ADMIN_PASSWORD_HASH = '5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8';
   ```

5. **Замени хеш** на свой:
   ```typescript
   const ADMIN_PASSWORD_HASH = 'ВАШ_НОВЫЙ_ХЕШ';
   ```

6. **Сохрани файл**

## 🎯 Требования к паролю

### Минимальные требования:
- ✅ Минимум 8 символов
- ✅ Латинские буквы и цифры

### Рекомендуется:
- ✅ Минимум 12 символов
- ✅ Заглавные и строчные буквы
- ✅ Цифры
- ✅ Специальные символы (!@#$%^&*)

### Примеры хороших паролей:
- `MySecure2024Pass!`
- `OKstudio#Admin99`
- `Kreuztal@Web2024`

### ❌ НЕ используй:
- `password`
- `123456`
- `admin`
- `qwerty`
- Простые слова из словаря

## 🔒 Как работает защита

### Двухуровневая аутентификация:

1. **Уровень 1: Пароль**
   - SHA-256 хеширование
   - Хранение в sessionStorage
   - Rate limiting (5 попыток)
   - Автоматическая блокировка

2. **Уровень 2: GitHub Token**
   - После ввода пароля
   - Проверка доступа к репозиторию
   - Безопасное хранение в localStorage

### Безопасность:
- ✅ Пароль никогда не хранится в открытом виде
- ✅ Только SHA-256 хеш
- ✅ Session-based аутентификация
- ✅ Автоматический logout при закрытии браузера
- ✅ Rate limiting против brute-force атак

## 🚀 Использование

### Вход в админ-панель:

1. Открой: `https://webstudio-ok.de/admin`

2. **Экран 1: Пароль**
   - Введи пароль
   - Нажми "ZUGANG GEWÄHREN"

3. **Экран 2: GitHub**
   - Введи GitHub Token
   - Введи Owner, Repo, Branch
   - Нажми "VERBINDEN"

4. **Готово!** Теперь ты в админ-панели

### Выход:

Нажми кнопку **Logout** (🚪) в правом верхнем углу.

Это очистит:
- ✅ Password session
- ✅ GitHub config
- ✅ Все данные аутентификации

## 🛡️ Дополнительная безопасность

### Рекомендации:

1. **Используй HTTPS**
   - Обязательно в production
   - Защищает от перехвата данных

2. **Регулярно меняй пароль**
   - Каждые 3-6 месяцев
   - После подозрительной активности

3. **Не делись паролем**
   - Каждому пользователю — свой пароль
   - Используй менеджер паролей

4. **Мониторь доступ**
   - Проверяй GitHub commits
   - Следи за изменениями

5. **Backup**
   - Храни пароль в безопасном месте
   - Используй менеджер паролей (1Password, Bitwarden)

## 🔧 Troubleshooting

### Забыл пароль?

1. Сгенерируй новый хеш:
   ```bash
   npx tsx generate-password-hash.ts НовыйПароль123
   ```

2. Замени хеш в `src/pages/Admin.tsx`

3. Пересобери проект:
   ```bash
   npm run build
   ```

4. Задеплой изменения

### Слишком много попыток?

Подожди 5 минут или:
1. Открой DevTools (F12)
2. Application → Session Storage
3. Удали `admin_auth`
4. Обнови страницу

### Пароль не работает?

1. Проверь, что хеш правильный:
   ```bash
   npx tsx generate-password-hash.ts ТвойПароль
   ```

2. Сравни с хешем в `Admin.tsx`

3. Убедись, что нет лишних пробелов
