## 🔐 Настройка безопасности

### 1. Смена пароля (ОБЯЗАТЕЛЬНО!)

**По умолчанию пароль**: `password`

⚠️ **ВАЖНО**: Смените пароль перед production!

```bash
# Генерация нового пароля
npm run password:generate
```

Следуйте инструкциям:
1. Введите новый пароль
2. Скопируйте сгенерированный хеш
3. Откройте `src/pages/Admin.tsx`
4. Найдите строку 32: `const ADMIN_PASSWORD_HASH = '...'`
5. Замените хеш на новый
6. Сохраните файл
7. Пересоберите проект: `npm run build`

### 2. GitHub Token

Создайте Personal Access Token на GitHub:

1. Откройте GitHub → Settings → Developer settings
2. Personal access tokens → Tokens (classic)
3. Generate new token (classic)
4. Выберите scope: **repo** (full control of private repositories)
5. Скопируйте токен (он показывается только один раз!)
6. Сохраните токен в безопасном месте

**Параметры для входа**:
- **Token**: Ваш GitHub Personal Access Token
- **Owner**: Ваш GitHub username
- **Repo**: Название репозитория (например: `studio-ok`)
- **Branch**: Ветка для commits (обычно `main`)

## 📦 Развертывание на Vercel

### Автоматическое развертывание

1. **Подключите репозиторий к Vercel**:
   ```bash
   # Если еще не подключено
   vercel
   ```

2. **Настройте переменные окружения** (если нужно):
   - В Vercel Dashboard → Settings → Environment Variables
   - Добавьте необходимые переменные

3. **Автоматический деплой**:
   - Каждый push в `main` ветку → автоматический деплой
   - Каждый commit из Admin панели → автоматический деплой

### Ручное развертывание

```bash
# Production деплой
vercel --prod

# Preview деплой
vercel
```

## 📦 Развертывание на Netlify

### Автоматическое развертывание

1. **Подключите репозиторий к Netlify**:
   - Netlify Dashboard → New site from Git
   - Выберите GitHub репозиторий
   - Branch: `main`
   - Build command: `npm run build`
   - Publish directory: `dist`

2. **Автоматический деплой**:
   - Каждый push в `main` → автоматический деплой
   - Каждый commit из Admin панели → автоматический деплой

### Ручное развертывание

```bash
# Установить Netlify CLI
npm install -g netlify-cli

# Деплой
netlify deploy --prod
```

## 🔄 Workflow

### Как работает система

```
┌─────────────────────────────────────────────────────────┐
│                    ADMIN PANEL                          │
│  (https://ваш-сайт.com/admin)                          │
└─────────────────────────────────────────────────────────┘
                         │
                         │ 1. Редактирование данных
                         ↓
┌─────────────────────────────────────────────────────────┐
│                  GITHUB REPOSITORY                      │
│  - Создается commit с изменениями                       │
│  - Обновляются JSON файлы                               │
└─────────────────────────────────────────────────────────┘
                         │
                         │ 2. Webhook
                         ↓
┌─────────────────────────────────────────────────────────┐
│              VERCEL / NETLIFY                           │
│  - Получает уведомление о новом commit                  │
│  - Запускает автоматический build                       │
│  - Деплоит новую версию сайта                           │
└─────────────────────────────────────────────────────────┘
                         │
                         │ 3. Обновление (1-3 минуты)
                         ↓
┌─────────────────────────────────────────────────────────┐
│                   LIVE WEBSITE                          │
│  (https://ваш-сайт.com)                                │
│  - Новые данные отображаются на сайте                   │
└─────────────────────────────────────────────────────────┘
```

### Время обновления

- **Сохранение в GitHub**: ~1-2 секунды
- **Webhook trigger**: ~5-10 секунд
- **Build и деплой**: ~1-2 минуты
- **Общее время**: ~2-3 минуты

## 🐛 Troubleshooting

### Проблема: "Token ungültig"

**Решение**:
1. Проверьте, что token имеет scope `repo`
2. Проверьте правильность owner/repo/branch
3. Создайте новый token
4. Убедитесь, что token не истек

### Проблема: "Fehler beim Speichern"

**Решение**:
1. Проверьте интернет соединение
2. Проверьте, что GitHub доступен
3. Проверьте права доступа к репозиторию
4. Проверьте консоль браузера на ошибки

### Проблема: Изменения не появляются на сайте

**Решение**:
1. Подождите 2-3 минуты (время деплоя)
2. Очистите кеш браузера (Ctrl+F5)
3. Проверьте GitHub - commit должен быть создан
4. Проверьте Vercel/Netlify Dashboard - деплой должен быть запущен

### Проблема: Build ошибки

**Решение**:
```bash
# Очистить node_modules и переустановить
rm -rf node_modules package-lock.json
npm install

# Очистить кеш
npm cache clean --force

# Попробовать снова
npm run build
```

### Проблема: Забыли пароль

**Решение**:
```bash
# Сгенерировать новый пароль
npm run password:generate

# Заменить хеш в src/pages/Admin.tsx (строка 32)
# Пересобрать и задеплоить
npm run build
vercel --prod  # или netlify deploy --prod
```

## 🔒 Безопасность в Production

### Checklist

- [ ] Пароль изменен с дефолтного
- [ ] GitHub Token имеет минимальные необходимые права
- [ ] HTTPS включен (автоматически на Vercel/Netlify)

3. **Не делитесь токеном** с другими людьми
4. **Используйте разные токены** для разных проектов
5. **Регулярно проверяйте GitHub Activity** на подозрительные commits

## 📝 Backup и восстановление

### Автоматический backup

GitHub автоматически хранит всю историю изменений:

```bash
# Просмотр истории
git log

# Откат к предыдущей версии
git revert <commit-hash>

# Восстановление файла
git checkout <commit-hash> -- path/to/file.json
```

### Ручной backup

```bash
# Клонировать репозиторий
git clone https://github.com/owner/repo.git backup-$(date +%Y%m%d)

# Или скачать ZIP
# GitHub → Code → Download ZIP
```

### Восстановление

1. Найдите нужный commit в GitHub
2. Откройте файл в этом commit
3. Скопируйте содержимое
4. Вставьте в Admin панель
5. Сохраните
