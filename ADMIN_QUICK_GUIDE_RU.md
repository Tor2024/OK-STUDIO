# Admin Panel - Краткое руководство

## 🚀 Быстрый старт

### 1. Вход в систему

1. Откройте `https://ваш-сайт.com/admin`
2. Введите пароль (по умолчанию: `password` - **ОБЯЗАТЕЛЬНО СМЕНИТЕ!**)
3. Введите GitHub Token и данные репозитория
4. Готово! Вы в Admin панели

### 2. Смена пароля

```bash
npm run password:generate
```

Следуйте инструкциям и замените хеш в `src/pages/Admin.tsx` (строка 32).

## 📚 Основные операции

### ПРОЕКТЫ (Portfolio)

#### Добавить проект:
1. Вкладка **PORTFOLIO**
2. Заполните форму слева:
   - **Titel**: Название проекта
   - **Kategorie**: Web Relaunch / System Dev / AI & Automation
   - **Bild-URL**: Полный URL изображения
   - **Abgeschlossen**: Формат "MRZ 2025" (заглавные буквы)
   - **Kurzbeschreibung**: Краткое описание для списка
   - **Volltext**: Полное описание с Markdown
   - **Reihenfolge**: Число (меньше = выше в списке)
3. Нажмите **SPEICHERN & DEPLOYEN**

#### Редактировать проект:
1. Нажмите синюю кнопку **Edit** рядом с проектом
2. Измените нужные поля
3. Нажмите **AKTUALISIEREN**

#### Удалить проект:
1. Нажмите красную кнопку **Delete**
2. Подтвердите удаление

---

### СТАТЬИ (Journal)

#### Добавить статью:
1. Вкладка **JOURNAL**
2. Заполните форму:
   - **Titel**: Заголовок статьи
   - **Datum**: Формат "Mai 2025" (первая буква заглавная)
   - **Tag**: DESIGN / KI / STRATEGIE / NEWS
   - **Autor**: Oleh Kalchenko (или другой автор)
   - **Inhalt**: Полный текст с Markdown
3. Используйте вкладку **VORSCHAU** для проверки
4. Нажмите **MARKDOWN HILFE** для справки по форматированию
5. Нажмите **PUBLIZIEREN & DEPLOYEN**

#### Markdown шпаргалка:
```markdown
## Большой заголовок
### Средний заголовок

**Жирный текст**
*Курсив*

- Список
- Пункт 2

> Цитата

[Ссылка](https://example.com)
```

---

### КЛИЕНТЫ (Kunden)

#### Добавить клиента:
1. Вкладка **KUNDEN**
2. Заполните:
   - **Name**: Название компании
   - **Website**: URL сайта (опционально)
   - **Reihenfolge**: Порядок отображения
3. Нажмите **HINZUFÜGEN & DEPLOYEN**

---

### FAQ

#### Добавить FAQ:
1. Вкладка **FAQ**
2. Заполните:
   - **Frage**: Вопрос
   - **Antwort**: Ответ (может быть длинным)
   - **Reihenfolge**: Порядок отображения
3. Нажмите **SPEICHERN & DEPLOYEN**

---

### НАСТРОЙКИ (Einstellungen)

1. Вкладка **EINSTELLUNGEN**
2. Измените нужные поля:
   - **SYSTEMSTATUS**: Статус сайта
   - **NODE LOCATION**: Локация сервера
   - **UPTIME**: Время работы
   - **ENGINE VERSION**: Версия системы
3. Нажмите **SPEICHERN & DEPLOYEN**

## 💡 Полезные советы

### Форматы данных:

| Поле | Формат | Пример |
|------|--------|--------|
| Дата проекта | MRZ YYYY | MRZ 2025 |
| Дата статьи | Mmm YYYY | Mai 2025 |
| URL изображения | https://... | https://images.unsplash.com/... |
| Порядок | Число | 1, 2, 3... |

### Рекомендации:

✅ **DO:**
- Используйте качественные изображения (минимум 1200px ширина)
- Пишите понятные описания
- Используйте Markdown для форматирования
- Проверяйте preview перед публикацией
- Сохраняйте резервные копии важных текстов

❌ **DON'T:**
- Не используйте слишком большие изображения (>2MB)
- Не забывайте про обязательные поля
- Не используйте специальные символы в названиях
- Не удаляйте данные без подтверждения

## 🔄 Как работает система

```
1. Вы редактируете данные в Admin панели
         ↓
2. Данные сохраняются в GitHub репозиторий
         ↓
3. GitHub отправляет webhook на Vercel/Netlify
         ↓
4. Сайт автоматически пересобирается
         ↓
5. Новые данные появляются на сайте (1-3 минуты)
```

## 🆘 Частые проблемы

### "Token ungültig"
- Проверьте, что token имеет scope `repo`
- Проверьте owner/repo/branch
- Создайте новый token на GitHub

### "Fehler beim Speichern"
- Проверьте интернет соединение
- Проверьте, что GitHub доступен
- Попробуйте еще раз через минуту

### Изменения не появляются на сайте
- Подождите 2-3 минуты (время деплоя)
- Очистите кеш браузера (Ctrl+F5)
- Проверьте GitHub - commit должен быть создан

### Забыли пароль
- Сгенерируйте новый хеш: `npm run password:generate`
- Замените хеш в `src/pages/Admin.tsx` (строка 32)
- Задеплойте изменения

## 🔐 Безопасность

### Важно:
- ⚠️ **СМЕНИТЕ ПАРОЛЬ** перед production!
- ⚠️ Не делитесь GitHub Token
- ⚠️ Используйте HTTPS
- ⚠️ Регулярно обновляйте пароль
- ⚠️ Не сохраняйте пароль в браузере на публичных компьютерах

### Logout:
- Всегда делайте logout после работы
- Особенно на публичных компьютерах
- Logout очищает и пароль, и GitHub token

## 📊 Структура данных

### Проекты и Статьи (оптимизированная):
- Каждый проект/статья = отдельный JSON файл
- Index файл = легкий список без полного контента
- При редактировании загружается полный файл
- При сохранении обновляются оба файла

### Клиенты и FAQ (простая):
- Все данные в одном JSON файле
- Быстрая загрузка и сохранение
